# Техническое задание для AI-агента: RadioCalc Single Page Application

## 1. Концепция и Стек
Необходимо создать **RadioCalc** — легковесное, быстрое и функциональное веб-приложение для радиолюбителей. Приложение должно работать полностью на стороне клиента (Client-Side), не требовать бэкенда, баз данных или ИИ. 

**Целевой хостинг:** Firebase Hosting (Free Tier).

### Стек технологий:
* **Фреймворк:** React (чистый клиентский билд без SSR/Next.js).
* **Сборщик:** Vite (для обеспечения максимальной скорости и минимального размера бандла).
* **Стилизация:** Tailwind CSS.
* **Иконки:** Lucide-React.

## 2. Ключевые инженерные допущения и запасы (Важно!)
В отличие от абстрактных математических формул, калькулятор должен выдавать **практические результаты**, применимые в реальной жизни и на печатных платах.

### А. Запас по мощности для резисторов
При расчете ограничительных или нагрузочных резисторов (например, для светодиодов) нельзя использовать резистор впритык к расчетной мощности:
* **Логика калькулятора:** После вычета рассеиваемой мощности ($P = I^2 \times R$), система должна автоматически закладывать **инженерный запас не менее 30–50%**.
* **Стандартизация номиналов:** Калькулятор должен подбирать ближайший **больший** стандартный шаг мощности выводных (выводных/SMD) резисторов: `0.125W`, `0.25W`, `0.5W`, `1W`, `2W`, `5W`.
* *Пример:* Если расчетная мощность составила 0.18W, с учетом запаса (+30% = 0.234W) приложение должно порекомендовать резистор минимум на `0.25W`.

### Б. Геометрия дорожек печатной платы (PCB Trace Width)
Для удобства проектирования прямо на плате, в калькулятор добавляется модуль расчета ширины дорожек.
* Расчет ведется по стандарту **IPC-2152** или **IPC-2221**.
* **Входные параметры:** Ток ($I$, Амперы), толщина медной фольги (стандарты: `18 мкм`, `35 мкм`, `70 мкм`), допустимый нагрев дорожки (обычно `10°C` или `20°C`), расположение (внутренний или внешний слой платы).
* **Выход:** Минимальная ширина дорожки в миллиметрах (mm) с рекомендацией округления в большую сторону для безопасной трассировки.

## 3. Функциональные модули приложения (Вкладки)

### Вкладка 1: Закон Ома и Мощность
* 4 взаимосвязанных поля: $U$, $I$, $R$, $P$.
* Ввод любых двух параметров мгновенно рассчитывает два оставшихся.

### Вкладка 2: Светодиод и Ограничительный резистор (с запасом)
* **Ввод:** Напряжение питания ($V_s$), падение напряжения на LED ($V_f$), рабочий ток LED ($I_f$).
* **Вывод:** 1. Точное расчетное сопротивление.
    2. Ближайший номинал из стандартного ряда **E24** или **E12**.
    3. Выделяемая мощность + **рекомендуемая мощность резистора с запасом**.

### Вкладка 3: Ширина дорожки платы (PCB Trace)
* Инструмент для безопасной разводки питания. Позволяет не сжечь плату при высоких токах.

### Вкладка 4: Маркировка компонентов
* Декодер цветовых колец резисторов (4-5 полос).
* Декодер SMD-кодов (3-4 цифры, EIA-96).

### Вкладка 5: Пассивные фильтры и Делители
* Расчет частоты среза RC/LC фильтров.
* Расчет плеч делителя напряжения ($R_1$, $R_2$).

## 4. UI/UX Требования
* **Тема:** Темная (Dark Mode) по умолчанию (удобно для глаз при пайке в плохо освещенном месте).
* **Адаптивность:** Mobile-first. Интерфейс должен идеально сидеть на экране смартфона.
* **Реактивность:** Изменение любого инпута мгновенно пересчитывает результат (без кнопок «Рассчитать»).
